CSS Selectors

CSS can select elements by their type, class and ID. Classes are meant to be reused over many elements. ID is meant to style only one element (will override styles of type classes so should be used sparingly).

Selector Example Description
Type p { Decides which elements style will be applied to. Matches the type of the element in the HTML doc. Often referred to by their opening tag name (therefore known as the tag name or element selector).
Universal * { Select all elements of any type. Can be used to reset default browser styling or all children of a parent element.
Class .brand { Selects a HTML element by its class.
ID #green { Selects HTML elements by their ID. Note: IDs can only have a single value and be used once per page.
Attributes [href] { Selects HTML elements with of the same type (based on their attribute or attribute value).

Specifity & Chaining

Name Description
Specifity Order by which browser decides which CSS styles will be displayed. Best practice is to style elements while using lowest degree of specifity so if an element needs a new style it is easy to override. ID is most specific, type is the least.
Chaining Combine multiple selectors to style more specific elements with more than one attribute. e.g. h1.special { would only select H1 elements with a 'special' class.
Descendent Combinator Select elements nested in other elements. e.g. in a list, descendents of <ul> can be selected.
Multiple Selectors Can add CSS styles to multiple selectors at once to be concise. Selectors need to be separated with a comma. e.g. h1, .menu {

Note: Adding more than one tag, class or ID to a CSS selector increases the specifity of the selector.
